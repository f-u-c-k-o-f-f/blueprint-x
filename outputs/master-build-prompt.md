Output 5: Master Build Prompt

Objective

Create the smallest implementation capable of solving the validated problem.

⸻

Inputs

* Discovery Summary
* Product Specification
* Architecture Recommendation
* Roadmap/Build Prompt

⸻

Requirements

* Minimize cost.
* Minimize infrastructure.
* Prefer local solutions.
* Prefer open-source solutions.
* Avoid premature optimization.
* Avoid unnecessary dependencies.

⸻

Build Rules

1. Implement MVP only.
2. Ignore future features.
3. Avoid scaling concerns.
4. Avoid enterprise patterns.
5. Prefer simple implementations.
6. Challenge every dependency.
7. Challenge every database.
8. Challenge every API.
9. Challenge every automation.

⸻

Deliverables

* Repository Structure
* Technical Design
* Implementation Tasks
* Build Order
* Validation Checklist
